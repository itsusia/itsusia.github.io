     171             :     needed to implement the expression for the rotation matrix derived over

     174             : 
     175             :     By default small angle approximations are used:
     176             :     The matrix elements are approximated by expanding the trigonometric
     177             :     functions to linear order (i.e. cos(x) = 1, sin(x) = x), and discarding
     178             :     products of second order.
     179             : 
     180             :     This was a useful hack when calculating by hand was the only option,
     181             :     but in general, today, should be avoided because:
     182             : 
     183             :     1. It does not save much computation time, as the rotation matrix
     184             :        is built only once and probably used many times (except when
     185             :        transforming spatio-temporal coordinates).
     186             : 
     187             :     2. The error induced may be too large for ultra high accuracy
     188             :        applications: the Earth is huge and the linear error is
     189             :        approximately the angular error multiplied by the Earth radius.
     190             : 
     191             :     However, in many cases the approximation is necessary, since it has
     192             :     been used historically: Rotation angles from older published datum
     193             :     shifts may actually be a least squares fit to the linearized rotation
     194             :     approximation, hence not being strictly valid for deriving the exact
     195             :     rotation matrix. In fact, most publicly available transformation
     196             :     parameters are based on the approximate Helmert transform, which is why
     197             :     we use that as the default setting, even though it is more correct to
     198             :     use the exact form of the equations.
     199             : 
     200             :     So in order to fit historically derived coordinates, the access to
     201             :     the approximate rotation matrix is necessary - at least in principle.
     202             : 
     203             :     Also, when using any published datum transformation information, one
     204             :     should always check which convention (exact or approximate rotation
     205             :     matrix) is expected, and whether the induced error for selecting
     206             :     the opposite convention is acceptable (which it often is).
     207             : 
     208             : 
     209             :     Sign conventions
     210             :     ----------------
     211             : 
     212             :     Take care: Two different sign conventions exist for the rotation terms.
     213             : 
     214             :     Conceptually they relate to whether we rotate the coordinate system
     215             :     or the "position vector" (the vector going from the coordinate system
     216             :     origin to the point being transformed, i.e. the point coordinates
     217             :     interpreted as vector coordinates).

     220             :     conventions is simply a matter of switching the sign of the rotation
     221             :     angles, which algebraically also translates into a transposition of
     222             :     the rotation matrix.
     223             : 
     224             :     Hence, as geodetic constants should preferably be referred to exactly
     225             :     as published, the "convention" option provides the ability to switch
